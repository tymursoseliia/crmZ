# 👑 Империя СКАМА - Генератор автодокументов

Система для автоматической генерации документов и управления финансами компании.

## 🚀 Возможности

### 📄 Генерация документов
- **TÜV Rheinland** - акты осмотра автомобилей
- **DEKRA** - акты осмотра автомобилей
- **Талоны очереди** - электронная очередь

### 💰 Финансовый модуль
- Управление операциями (растаможка, добив)
- Учет расходов (персональные, технические, постоянные, общие)
- Управление сотрудниками и командами
- Автоматический расчет зарплат
- Расчет по периодам
- Аналитика и отчеты
- Экспорт в Excel

### 📊 CRM - Воронка продаж (НОВОЕ!)
- **История статусов** - отслеживание всех изменений
- **Воронка продаж** - визуализация с конверсией между этапами
- **Анализ срезов** - откуда лиды переходят в "Потеряно"
- **Среднее время** - сколько дней на каждом этапе
- **Статистика команд** - сравнение эффективности
- **Real-time обновления** - синхронизация через Supabase

### 🏢 Управление компаниями
- База данных компаний
- Документы компаний (уставы, свидетельства, доверенности)

### 🔐 Безопасность
- Двухуровневая система авторизации
- Защита паролем для входа на сайт
- Дополнительный пароль для доступа к финансам
- Приватное хранение данных

## 🛠️ Технологии

- **Next.js 15** - React фреймворк
- **TypeScript** - типизация
- **Tailwind CSS** - стилизация
- **shadcn/ui** - UI компоненты
- **Supabase** - облачная база данных (опционально)
- **Recharts** - графики и диаграммы

- **xlsx** - экспорт в Excel

## 📦 Установка и запуск

### Локальная разработка

1. **Клонируйте репозиторий:**
```bash
git clone https://github.com/gennadiyyasnov-maker/skamsite.git
cd skamsite
```

2. **Установите зависимости:**
```bash
npm install
# или
bun install
```

3. **Настройте переменные окружения (опционально):**
```bash
cp .env.example .env.local
```

4. **Запустите dev сервер:**
```bash
npm run dev
# или
bun dev
```

5. **Откройте в браузере:**
```
http://localhost:3000
```

### 🗄️ Настройка базы данных Supabase

Для подключения облачной базы данных следуйте инструкции:

**👉 [SUPABASE_SETUP.md](./SUPABASE_SETUP.md)** - Полная инструкция по настройке Supabase

Без Supabase данные будут храниться в `localStorage` браузера.

## 📊 Финансовый модуль - как работает

### Команды и сотрудники
- **Команда Вохи** - менеджеры + тимлид Воха
- **Команда Зета** - менеджеры + тимлид Зет
- **Офис** - Жигуль, Клыча, клоузеры
- **IT отдел** - разработчики с фиксированной ставкой
- **Особые роли** - Звук (2% от общей прибыли), Фотошоп

### Расчет зарплат
- **Менеджеры:** % от растаможек и добивов (не меньше минималки)
- **Клоузеры:** 5% от добивов
- **Тимлиды:** 5% от прибыли команды
- **Звук:** 2% от общей прибыли обеих команд
- **IT отдел:** фиксированная ставка (делится 50/50 между командами)

### Распределение расходов
- **Персональные** - относятся к конкретной команде
- **Технические** - делятся 50/50 между командами
- **Постоянные** - делятся 50/50 между командами
- **Общие** - делятся 50/50 между командами
