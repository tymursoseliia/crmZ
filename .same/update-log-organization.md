# 🔄 Обновление: Организация системы статусов
**Дата:** 04.02.2026 (Часть 2)

## 🎯 Проблема

Изначально в CRM можно было:
- ❌ Изменять любой статус на любой
- ❌ Потерять лида без указания причины
- ❌ Создать оплату без суммы
- ❌ Непонятно кто за что отвечает

**Результат:** Невозможно понять ПОЧЕМУ теряются лиды и ГДЕ проблема.

## ✨ Решение

Создана **организованная система управления статусами** с четкими правилами и валидацией.

## 📋 Что изменилось

### 1. **Новый компонент LeadStatusChanger**
Специальный диалог для изменения статуса с:
- ✅ Валидацией переходов между статусами
- ✅ Обязательными полями в зависимости от статуса
- ✅ Понятным интерфейсом с подсказками

### 2. **Правила переходов**
```
Можно:
✅ Перейти на следующий статус
✅ Перейти в "Потеряно" с любого этапа
✅ Остаться на текущем статусе

Нельзя:
❌ Перепрыгнуть через этап
❌ Вернуться назад
❌ Изменить финальные статусы (Завершено/Потеряно)
```

### 3. **Обязательные поля**

**При оплате:**
```typescript
status: 'payment_customs' | 'payment_car' | 'payment_recycling' | ...
ТРЕБУЕТСЯ: amount (сумма в рублях)
```

**При потере:**
```typescript
status: 'lost'
ТРЕБУЕТСЯ:
  - lostReason (причина из списка)
  - lostReasonText (подробное описание)
```

### 4. **Категории причин потери**
```typescript
type LostReason =
  | 'high_price'       // Высокая цена
  | 'long_wait'        // Долгое ожидание
  | 'found_competitor' // Нашел конкурента
  | 'changed_mind'     // Передумал
  | 'no_money'         // Нет денег
  | 'no_response'      // Не отвечает
  | 'other';           // Другое
```

### 5. **Новая аналитика причин потери**
Теперь в разделе "Аналитика" есть карточка:
```
📊 Детальный анализ причин потери:
├─ Высокая цена: 5 лидов (50%)
├─ Долгое ожидание: 3 лида (30%)
└─ Не отвечает: 2 лида (20%)
```

## 🔄 Как это работает

### Сценарий 1: Тимлид выдает реквизиты
```
1. Открывает лида со статусом "Договор сделан"
2. Нажимает "Изменить статус"
3. Видит доступные варианты:
   - Договор сделан (текущий)
   - Дал реквизиты (следующий)
   - Потеряно
4. Выбирает "Дал реквизиты"
5. Добавляет комментарий: "Выдал реквизиты по email"
6. Сохраняет
```

### Сценарий 2: Клиент оплатил растаможку
```
1. Открывает лида со статусом "Дал реквизиты"
2. Нажимает "Изменить статус"
3. Выбирает "Оплата за растаможку"
4. ОБЯЗАТЕЛЬНО указывает сумму: 500000 ₽
5. Добавляет комментарий: "Платеж поступил 04.02"
6. Сохраняет
```

### Сценарий 3: Лид потерян
```
1. Открывает лида на любом этапе
2. Нажимает "Изменить статус"
3. Выбирает "Потеряно"
4. ОБЯЗАТЕЛЬНО выбирает причину: "Высокая цена"
5. ОБЯЗАТЕЛЬНО пишет подробности:
   "Сказал что цена выше чем у конкурента на 100к"
6. Сохраняет
```

## 📊 Польза для аналитики

### До изменений:
```
Анализ срезов:
Оплата за растаможку → 5 лидов потеряно
```
Непонятно ПОЧЕМУ.

### После изменений:
```
Анализ срезов:
Оплата за растаможку → 5 лидов потеряно

Детальный анализ причин:
├─ Высокая цена: 3 лида (60%)
├─ Нет денег: 1 лид (20%)
└─ Долгое ожидание: 1 лид (20%)
```
**Вывод:** Проблема в цене! Нужно пересмотреть.
