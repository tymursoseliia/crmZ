## ✅ ЧТО БЫЛО ВОССТАНОВЛЕНО

### 1. Функция `closePeriod` (строки 192-222)

```typescript
const closePeriod = async () => {
  if (!currentPeriod) return;
  if (currentPeriod.is_closed) {
    alert('⚠️ Период уже закрыт');
    return;
  }

  if (!confirm(`Закрыть период "${formatDate(currentPeriod.start_date)} - ${formatDate(currentPeriod.end_date)}"?\n\n⚠️ После закрытия редактирование будет НЕВОЗМОЖНО!`)) {
    return;
  }

  try {
    if (supabase) {
      // Закрываем период в БД
      await supabase
        .from('telegram_periods')
        .update({ is_closed: true })
        .eq('id', currentPeriod.id);
    }

    // Обновляем локальное состояние
    const updatedPeriod = { ...currentPeriod, is_closed: true };
    setCurrentPeriod(updatedPeriod);
    setPeriods(prev => prev.map(p => p.id === currentPeriod.id ? updatedPeriod : p));

    alert(`✅ Период закрыт!\n\nРедактирование данных больше невозможно.`);
  } catch (error) {
    console.error('[Telegram] Ошибка закрытия периода:', error);
    alert('❌ Ошибка закрытия периода');
  }
};
```

**Функциональность:**
- Проверяет, не закрыт ли период уже
- Показывает подтверждающее окно с предупреждением
- Обновляет поле `is_closed` в Supabase таблице `telegram_periods`
- Обновляет локальное состояние компонента
- Показывает уведомление об успехе/ошибке

## 🔄 WORKFLOW ИСПОЛЬЗОВАНИЯ

### Сценарий 1: Закрытие периода

1. **Пользователь** работает с текущим периодом
2. **Заполняет** данные по лидам для всех менеджеров
3. **Проверяет** итоговые суммы
4. **Нажимает** "Закрыть период"
5. **Подтверждает** в диалоге
6. **Система** обновляет БД и блокирует редактирование
7. **Пользователь** видит бейдж "Период закрыт"

### Сценарий 2: Попытка редактирования закрытого периода

1. **Пользователь** выбирает закрытый период из списка
2. **Видит** красный бейдж "Период закрыт"
3. **Пытается** кликнуть на ячейку
4. **Ничего не происходит** - ячейка disabled
5. **Видит** подсказку: "Период закрыт для редактирования"

### Сценарий 3: Создание нового периода

1. **Пользователь** закрывает текущий период
2. **Нажимает** "Создать новый период"
3. **Система** создает период на следующие 2 недели
4. **Новый период** становится текущим и открытым (is_closed = false)
5. **Старый период** остается в списке со статусом [закрыт]
