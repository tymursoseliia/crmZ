# 🎯 Фиксация этапа среза - Важное обновление

## 📋 Проблема

**Было:**
```
Лид на этапе: "Оплата за растаможку"
Переводим в: "Потеряно"
Система фиксирует: Срез на этапе "Оплата за растаможку"
```

**Но реально:**
```
Клиент потерялся еще на этапе "Дал реквизиты"
(долго ждал документы, за это время ушел к конкуренту)
Мы просто не сразу это поняли
```

**Результат:** Неточная аналитика! ❌

## ✅ Решение

Теперь при переводе лида в "Потеряно" нужно **обязательно указать**:
1. **На каком этапе произошел срез** (где реально потерялся клиент)
2. **Причину потери** (почему это произошло)
3. **Подробности** (детальное описание)

## 🔄 Как это работает

### Новое поле в диалоге

При выборе статуса "Потеряно" появляется:

```
┌─────────────────────────────────────────────┐
│ На каком этапе произошел срез? *            │
│ ┌─────────────────────────────────────────┐ │
│ │ [▼] Дал реквизиты                        │ │
│ └─────────────────────────────────────────┘ │
│ Выберите этап, на котором реально           │
│ потерялся клиент                            │
└─────────────────────────────────────────────┘
```

### Доступные варианты:
- Договор сделан
- Дал реквизиты
- Оплата за растаможку
- Оплата за авто
- Оплата за утиль
- Оплата за госпошлину
- Оплата за залоговый платеж
- Оплата прочее
- Завершено

## 📊 Примеры использования

### Пример 1: Срез на этапе выдачи реквизитов

**Ситуация:**
- Договор подписан 01.02
- Реквизиты выданы 06.02 (прошло 5 дней!)
- Клиент за это время нашел конкурента
- 10.02 мы поняли что лид потерян

**Как фиксировать:**
```
Текущий статус: Дал реквизиты
Новый статус: Потеряно

┌─ На каком этапе произошел срез?
│  Дал реквизиты ✅
│
├─ Причина: Долгое ожидание
│
└─ Подробности:
   "Ждал реквизиты 5 дней (01.02 - 06.02).
    За это время нашел другую компанию, которая
    предоставила реквизиты в тот же день."
```

### Пример 2: Срез на этапе оплаты

**Ситуация:**
- Лид дошел до "Оплата за растаможку"
- Увидел сумму 500 000 ₽
- Сказал что это дорого
- Попросил время подумать
- Больше не вышел на связь

**Как фиксировать:**
```
Текущий статус: Оплата за растаможку
Новый статус: Потеряно

┌─ На каком этапе произошел срез?
│  Оплата за растаможку ✅
│
├─ Причина: Высокая цена
│
└─ Подробности:
   "Сумма 500 000 ₽. Сказал что у конкурента
    на 50к дешевле. Попросил время подумать,
    больше не отвечает на звонки."
```

### Пример 3: Поздно обнаруженный срез

**Ситуация:**
- Лид формально на этапе "Оплата за авто"
- Но 2 недели назад перестал отвечать
- Только сейчас мы это заметили
- На самом деле потерялся раньше

**Как фиксировать:**
```
Текущий статус: Оплата за авто
Новый статус: Потеряно

┌─ На каком этапе произошел срез?
│  Оплата за растаможку ✅ (не "Оплата за авто"!)
│
├─ Причина: Не отвечает
│
└─ Подробности:
   "Последний контакт был 20.01 на этапе 'Оплата
    за растаможку'. После этого не отвечал на звонки
    2 недели. Только 04.02 поняли что лид потерян."
```

## 📈 Влияние на аналитику

### До изменений:
```
Анализ срезов:
├─ Оплата за растаможку → 3 лида
├─ Оплата за авто → 2 лида
└─ Дал реквизиты → 0 лидов

Непонятно где реальная проблема!
```

### После изменений:
```
Анализ срезов (по lostAtStage):
├─ Дал реквизиты → 5 лидов ⚠️ ПРОБЛЕМА!
├─ Оплата за растаможку → 0 лидов
└─ Оплата за авто → 0 лидов

Детальный анализ причин:
Дал реквизиты:
├─ Долгое ожидание: 3 лида (60%)
└─ Нашел конкурента: 2 лида (40%)

ВЫВОД: Нужно ускорить выдачу реквизитов!
```

## 🎯 Правила фиксации

### Выбирайте этап где:
✅ Клиент РЕАЛЬНО начал терять интерес
✅ Произошло событие, из-за которого клиент ушел
✅ Был последний конструктивный контакт

### НЕ выбирайте этап где:
❌ Лид формально находится сейчас
❌ Вы просто поняли что он потерян
❌ Прошло много времени без контакта

### Примеры правильного выбора:

**Правильно ✅:**
```
Лид на: Оплата за авто
Срез на: Дал реквизиты
Причина: Долгое ожидание (ждал реквизиты 7 дней)
```

**Неправильно ❌:**
```
Лид на: Оплата за авто
Срез на: Оплата за авто
Причина: Не отвечает
(Но реально он перестал отвечать еще когда ждал реквизиты!)
```

## 💡 Советы

### 1. Анализируйте весь путь клиента
Перед фиксацией срезапройдитесь по истории:
- Когда был последний контакт?
- Что могло пойти не так?
- На каком этапе появились проблемы?

### 2. Будьте честными
Если проблема была на этапе "Дал реквизиты" (медленно работали),
не сваливайте на "Не отвечает" на текущем этапе.

### 3. Подробные заметки помогают
Описывайте детально:
- Даты событий
- Что говорил клиент
- Какие были задержки
- Что можно было сделать лучше

### 4. Используйте для улучшений
Регулярно смотрите аналитику срезов и:
- Выявляйте проблемные этапы
- Ускоряйте процессы
- Улучшайте коммуникацию
