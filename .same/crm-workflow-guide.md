# 🔄 Организация работы с лидами - Workflow Guide

## 📋 Обзор

Правильно организованная система фиксации статусов лидов - ключ к эффективной аналитике. Этот документ описывает как правильно работать с лидами в CRM.

## 👥 Роли и обязанности

### 1. **Документалист**
**Задачи:**
- Создает нового лида после заключения договора
- Заполняет базовую информацию (ФИО, телефоны, email)
- Устанавливает начальный статус: **"Договор сделан"**
- Назначает ответственного менеджера/тимлида

**Что НЕ делает:**
- Не меняет статусы (только создание)
- Не удаляет лидов без согласования

### 2. **Тимлид (ТЛ)**
**Задачи:**
- Управляет всеми статусами своих лидов
- Выдает реквизиты → меняет на **"Дал реквизиты"**
- Отслеживает оплаты и обновляет статусы
- Указывает причину при потере лида
- Следит за актуальностью сумм

**Полномочия:**
- Полный доступ к изменению статусов
- Может переводить лида в "Потеряно" с указанием причины
- Редактирует информацию о лиде

### 3. **Владелец**
**Задачи:**
- Анализирует общую статистику
- Выявляет проблемные этапы
- Принимает решения по оптимизации

**Доступ:**
- Просмотр всей аналитики
- Сравнение эффективности команд

## 📊 Этапы воронки продаж

### Последовательность статусов:

```
1. Договор сделан (начальный)
   ↓
2. Дал реквизиты
   ↓
3. Оплата за растаможку
   ↓
4. Оплата за авто
   ↓
5. Оплата за утиль
   ↓
6. Оплата за госпошлину
   ↓
7. Оплата за залоговый платеж
   ↓
8. Оплата прочее
   ↓
9. Завершено (финал ✅)

В любой момент → Потеряно (финал ❌)
```

### Важные правила:

✅ **Можно:**
- Перейти на следующий статус
- Перейти в "Потеряно" с любого этапа
- Остаться на текущем статусе

❌ **Нельзя:**
- Перепрыгнуть через статус
- Вернуться назад
- Изменить статус "Завершено" или "Потеряно"

## 🔄 Процесс изменения статуса

### Шаг 1: Открыть диалог
На карточке лида нажать **"Изменить статус"**

### Шаг 2: Выбрать новый статус
Доступны только:
- Текущий статус (остаться)
- Следующий по порядку
- "Потеряно"

### Шаг 3: Заполнить обязательные поля

**Если выбрана оплата:**
- ✅ Обязательно указать **сумму в рублях**

**Если выбрано "Потеряно":**
- ✅ Обязательно выбрать **причину**:
  - Высокая цена
  - Долгое ожидание
  - Нашел конкурента
  - Передумал
  - Нет денег
  - Не отвечает
  - Другое
- ✅ Обязательно написать **подробности**

### Шаг 4: Добавить комментарий (опционально)
Любая дополнительная информация

### Шаг 5: Сохранить
Система автоматически:
- Запишет изменение в историю
- Обновит время изменения
- Сохранит в базу данных

## 📝 Примеры правильной работы

### Пример 1: Выдача реквизитов
```
Статус: Договор сделан
Действие: ТЛ выдал реквизиты клиенту

Шаги:
1. Открыть карточку лида
2. Нажать "Изменить статус"
3. Выбрать "Дал реквизиты"
4. Добавить комментарий: "Выдал реквизиты по email"
5. Сохранить
```

### Пример 2: Оплата растаможки
```
Статус: Дал реквизиты
Действие: Клиент оплатил растаможку

Шаги:
1. Открыть карточку лида
2. Нажать "Изменить статус"
3. Выбрать "Оплата за растаможку"
4. Указать сумму: 500000 ₽
5. Добавить комментарий: "Оплата поступила 04.02.2026"
6. Сохранить
```

### Пример 3: Потеря лида
```
Статус: Оплата за растаможку
Действие: Клиент не вышел на связь

Шаги:
1. Открыть карточку лида
2. Нажать "Изменить статус"
3. Выбрать "Потеряно"
4. Причина: "Не отвечает"
5. Подробности: "Не отвечает на звонки 3 дня, не приехал на встречу"
6. Сохранить
```

## ⚠️ Типичные ошибки

### ❌ Ошибка 1: Пропуск этапов
```
Было: Договор сделан
Стало: Оплата за авто
НЕПРАВИЛЬНО! Нужно пройти через "Дал реквизиты" и "Оплата за растаможку"
```

### ❌ Ошибка 2: Отсутствие причины потери
```
Статус: Потеряно
Причина: не указана
НЕПРАВИЛЬНО! Обязательно указывать причину для анализа
```

### ❌ Ошибка 3: Отсутствие суммы
```
Статус: Оплата за растаможку
Сумма: не указана
НЕПРАВИЛЬНО! При оплате обязательно указывать сумму
```

## 📊 Как это влияет на аналитику

### 1. **Воронка продаж**
Показывает на каких этапах скапливаются лиды
- Если много на "Дал реквизиты" → возможно долго готовим документы
- Если много на "Оплата растаможки" → возможно высокая цена

### 2. **Анализ срезов**
Показывает откуда лиды уходят в "Потеряно"
- "Оплата растаможки → Потеряно: 50%" → проблема с ценой
- "Дал реквизиты → Потеряно: 30%" → проблема с документами

### 3. **Причины потери**
Детализация почему теряются лиды
- "Высокая цена: 40%" → нужно пересмотреть ценообразование
- "Долгое ожидание: 30%" → ускорить процессы

### 4. **Среднее время**
Сколько дней лиды проводят на каждом этапе
- Если долго → оптимизировать процесс
- Если быстро → эффективная работа

## 💡 Советы по использованию

### Совет 1: Заметки имеют значение
Хорошие заметки помогают команде:
```
❌ Плохо: "Оплатил"
✅ Хорошо: "Оплата поступила 04.02, сумма 500 000 ₽, платеж #12345"
```

### Совет 2: Детальные причины потери
```
❌ Плохо: "Не отвечает"
✅ Хорошо: "Не отвечает на звонки 5 дней, последний контакт 01.02, обещал оплатить но пропал"
```

### Совет 3: Своевременное обновление
Обновляйте статусы в тот же день когда произошло изменение. Это важно для:
- Точной аналитики времени на этапах
- Актуальной картины воронки
- Правильных прогнозов
