# Руководство по деплою на Netlify с Supabase

## Что сделано

Финансовая CRM система теперь использует **Supabase** (облачная PostgreSQL база данных) вместо localStorage.

### Преимущества:
- ✅ Данные хранятся в облаке, доступны с любого устройства
- ✅ Real-time синхронизация между пользователями
- ✅ Автоматические бэкапы БД
- ✅ Работает на Netlify без дополнительной настройки
- ✅ Бесплатный тарий Supabase: до 500 MB БД, 50 000 запросов/мес
- ✅ Автоматический fallback на localStorage если Supabase недоступен

## Шаги для деплоя

### 1. Настройка Supabase

**Внимательно следуйте инструкциям в файле `.same/supabase-setup.md`:**

1. Создайте проект на https://supabase.com (бесплатно)
2. Выполните SQL скрипт из инструкции для создания таблиц
3. Скопируйте `Project URL` и `anon public` ключ

### 2. Настройка переменных окружения в Netlify

1. Откройте ваш проект на Netlify Dashboard
2. Перейдите в **Site settings** → **Environment variables**
3. Добавьте переменные:
   ```
   NEXT_PUBLIC_SUPABASE_URL = https://ваш-проект.supabase.co
   NEXT_PUBLIC_SUPABASE_ANON_KEY = eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...
   ```
4. Нажмите **Save**

### 3. Деплой на Netlify

Два способа:

#### Вариант А: Через GitHub (рекомендуется)

1. Закоммитьте код в GitHub:
   ```bash
   git add .
   git commit -m "Add Supabase integration"
   git push
   ```

2. На Netlify:
   - **Sites** → **Add new site** → **Import an existing project**
   - Выберите ваш GitHub репозиторий
   - Build settings уже настроены в `netlify.toml`
   - Нажмите **Deploy**

#### Вариант Б: Через Same (быстро)

Same автоматически деплоит на Netlify:
- Проект уже подключен к Netlify
- Переменные окружения нужно добавить вручную (шаг 2)
- После обновления кода Netlify автоматически переделоит

### 4. Проверка деплоя

1. Откройте деплой на Netlify
2. Перейдите на `/finances`
3. Вверху страницы должен быть зеленый индикатор: **"✅ Supabase БД подключена"**
4. Добавьте тестовую операцию или сотрудника
5. Откройте сайт в другом браузере - данные должны синхронизироваться

### 5. Работа без Supabase (опционально)

Если вы НЕ хотите настраивать Supabase:
- Приложение автоматически использует localStorage
- Будет показан желтый индикатор: **"⚠️ Используется localStorage"**
- Данные будут сохраняться только в браузере пользователя

## Мониторинг Supabase

### Просмотр данных
1. Откройте Supabase Dashboard
2. **Table Editor** → выберите таблицу (employees, operations, expenses)
3. Видите все данные в реальном времени

### Бэкапы
- **Database** → **Backups**
- Автоматические ежедневные бэкапы (хранятся 7 дней на бесплатном тарифе)

### Логи
- **Logs** → **Postgres Logs**
- Видите все запросы к БД

## Безопасность

⚠️ **Текущая настройка для разработки/тестирования:**
- Анонимный доступ разрешен к данным
- Для production рекомендуется настроить Supabase Auth

**Для production:**
1. Включите Supabase Authentication
2. Обновите Row Level Security политики
3. Добавьте аутентификацию пользователей

Подробнее: https://supabase.com/docs/guides/auth

## Лимиты бесплатного тарифа Supabase

- **База данных:** до 500 MB
- **API запросы:** 50,000 в месяц
- **Bandwidth:** 2 GB/мес
- **Storage:** 1 GB

Для вашего проекта этого **более чем достаточно**.

## Troubleshooting

### Индикатор показывает localStorage
**Проблема:** Желтый индикатор вместо зеленого

**Решение:**
1. Проверьте переменные окружения в Netlify
2. Убедитесь что имена переменных **точно** соответствуют:
   - `NEXT_PUBLIC_SUPABASE_URL`
   - `NEXT_PUBLIC_SUPABASE_ANON_KEY`
3. Переделой сайт: **Deploys** → **Trigger deploy** → **Clear cache and deploy**

### Ошибка "Failed to fetch" в консоли
**Проблема:** Запросы к Supabase блокируются

**Решение:**
1. Проверьте RLS политики в Supabase (должны разрешать анонимный доступ)
2. Проверьте URL проекта (должен начинаться с `https://`)
3. Откройте Network в DevTools и проверьте CORS

### Данные не синхронизируются
**Проблема:** Изменения не видны в других браузерах

**Решение:**
1. Убедитесь что индикатор зеленый (Supabase подключен)
2. Проверьте данные в Supabase Table Editor
3. Откройте консоль браузера на ошибки
